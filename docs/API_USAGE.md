# College Project Backend API Guide

This document explains how to use the backend APIs from Postman and frontend apps.

## 1) Base Setup

- Base URL: `https://college-project-backend-r7f9.onrender.com`
- Content type: `application/json`
- Auth: currently **no authentication middleware is enabled**

Example header:

```http
Content-Type: application/json
```

## 2) Postman Setup

1. Create a Postman environment.
2. Add variable `baseUrl = https://college-project-backend-r7f9.onrender.com`.
3. For every request, use URL format: `{{baseUrl}}/api/...`.
4. Use `Body -> raw -> JSON` for POST/PUT/PATCH.

## 3) Common Response Notes

- IDs are generated by backend (for example: `adm_*`, `stu_*`, `prj_*`, `doc_*`, etc).
- Keep generated IDs from one API response and use them in dependent APIs.
- Validation/business errors return JSON like:

```json
{
  "timestamp": "2026-02-24T14:00:00Z",
  "status": 400,
  "error": "Bad Request",
  "message": "Student mail already exists",
  "path": "/api/students"
}
```

## 4) Enum Values You Must Use

- `StudentEnrollStatus`: `ENROLLED`, `PENDING`, `REJECTED`
- `SupervisorEnrollStatus`: `ACTIVE`, `INACTIVE`
- `StageStatus`: `SYNOPSIS`, `PROGRESS1`, `PROGRESS2`, `FINAL`
- `SubmissionStatus`: `PENDING`, `APPROVED`, `REJECTED`, `REVISION`
- `CommentByRole`: `SUPERVISOR`, `ADMIN`
- `RequestStatus`: `PENDING`, `APPROVED`, `REJECTED`

## 5) Recommended End-to-End Workflow

### Step 1: Create Admin

`POST {{baseUrl}}/api/admins`

```json
{
  "name": "System Admin",
  "mail": "admin@college.edu",
  "password": "admin123",
  "department": "CSE"
}
```

Save `adminId` from response.

### Step 2: Create Form

`POST {{baseUrl}}/api/forms`

```json
{
  "accessBranch": "CSE",
  "accessBatch": "2022",
  "jsonOfFields": "[{\"name\":\"problemStatement\",\"type\":\"text\"}]",
  "createdBy": "<adminId>"
}
```

Save `formId`.

### Step 3: Create Students

`POST {{baseUrl}}/api/students`

```json
{
  "name": "Akshat Rai",
  "mail": "akshat@college.edu",
  "password": "student123",
  "rollNo": "CSE22001",
  "branch": "CSE",
  "batch": "2022",
  "enrollStatus": "ENROLLED"
}
```

Create more students if needed. Save their `studentId` values.

### Step 4: Create Supervisor

`POST {{baseUrl}}/api/supervisors`

```json
{
  "name": "Dr. Sharma",
  "mail": "sharma@college.edu",
  "password": "supervisor123",
  "branch": "CSE",
  "enrollStatus": "ACTIVE"
}
```

Save `supervisorId`.

### Step 5: Create Team

`POST {{baseUrl}}/api/teams`

```json
{
  "leaderId": "<studentId-leader>",
  "teamMemberIds": ["<studentId-member1>", "<studentId-member2>"],
  "teamCompleteStatus": true
}
```

Save `teamId`.

### Step 6: Create Project (auto-creates document if documentId not passed)

`POST {{baseUrl}}/api/projects`

```json
{
  "teamId": "<teamId>",
  "formId": "<formId>",
  "supervisorId": "<supervisorId>",
  "projectTitle": "AI Attendance System",
  "projectDescription": "Face recognition based attendance project",
  "status": "pending",
  "stageStatus": "SYNOPSIS",
  "supervisorAssignedBy": "<adminId>"
}
```

Save `projectId` and `documentId`.

### Step 7: Get document by project

`GET {{baseUrl}}/api/documents/by-project/<projectId>`

Use returned `documentId` for submissions.

### Step 8: Create stage submission

Example for synopsis:

`POST {{baseUrl}}/api/submissions/synopsis`

```json
{
  "documentId": "<documentId>",
  "comment": "Initial synopsis upload"
}
```

Similarly:

- `POST /api/submissions/progress1`
- `POST /api/submissions/progress2`
- `POST /api/submissions/final`

### Step 9: Review submission (supervisor/admin)

`PATCH {{baseUrl}}/api/submissions/synopsis/<submissionId>/review`

```json
{
  "comment": "Looks good",
  "status": "APPROVED",
  "commentByRole": "SUPERVISOR",
  "commentById": "<supervisorId>"
}
```

### Step 10: Update project scores/stage

`PUT {{baseUrl}}/api/projects/<projectId>`

```json
{
  "stageStatus": "PROGRESS1",
  "synopsisScore": "18",
  "stagesScoreOutOf": "20",
  "status": "in_progress"
}
```

### Step 11: Leaderboard entry

`POST {{baseUrl}}/api/leaderboard`

```json
{
  "teamId": "<teamId>",
  "supervisorId": "<supervisorId>",
  "projectId": "<projectId>",
  "rank": 1,
  "score": 88,
  "updateStatus": true
}
```

## 7) Frontend Integration Pattern

Use a shared API client and keep `baseURL` configurable.

Example (Axios):

```ts
import axios from "axios";

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || "http://localhost:8080",
  headers: { "Content-Type": "application/json" }
});

export const createStudent = (payload: any) => api.post("/api/students", payload);
export const createTeam = (payload: any) => api.post("/api/teams", payload);
export const createProject = (payload: any) => api.post("/api/projects", payload);
```

For production, set `VITE_API_BASE_URL` (or your framework equivalent) to your deployed backend URL.
